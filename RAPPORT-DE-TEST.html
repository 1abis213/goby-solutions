1. Bugs critiques

Critique Bouton « Continuer sur le site » — texte français hardcoded
Ligne 2665
Le bouton #popup-continue affiche "Continuer sur le site" en dur au lieu d'utiliser data-i18n="popupContinue". En mode anglais, le bouton affichera du français. En mode français, ça fonctionne par accident.
Correction : Ajouter data-i18n="popupContinue" au bouton et retirer le texte inline.
Critique Messages de validation du formulaire jamais traduits
Lignes 2206-2238
Les fonctions validateName(), validateEmail(), validatePhone(), validateMessage() retournent toutes des messages d'erreur en anglais hardcoded. Même si l'utilisateur passe en français, les erreurs restent en anglais.
Correction : Utiliser un système de traduction pour les messages de validation (clé i18n ou objet de traduction).
Critique Boutons « Open » / « Closes soon » — badges d'heures jamais traduits
Ligne 2335
b.textContent = closingSoon ? 'Closes soon' : 'Open'; — Ces textes sont en anglais et ne changent jamais avec la langue.
Correction : Utiliser les clés i18n ou un objet de traduction pour les badges.
Critique Les jours des heures d'ouverture ne sont jamais traduits
Lignes 1748-1775 (pied de page)
Les noms de jours (Sunday, Monday, etc.) dans la section heures du pied de page sont du texte brut, sans data-i18n. En français, ils restent affichés en anglais.
Correction : Ajouter des balises span avec data-i18n pour chaque jour, ou traduire via JS.
Critique Fermeture de tous les popups — un clic sur un lien ferme aussi les autres
Ligne 2750
document.querySelectorAll('.popup-overlay a').forEach(...) ajoute closePopup (fermeture du popup téléphonique) à TOUS les liens de TOUS les popups. Cliquer sur un lien dans le popup WhatsApp, les popups légaux, ou tout autre popup ferme aussi le popup téléphonique.
Correction : Cibler uniquement les liens dans #popup-overlay, pas tous les .popup-overlay a.
Critique Scrollbar du carrousel d'avis visible sur Chrome/Safari/Edge
Ligne 979
Le style inline utilise scrollbar-width:none (Firefox) et -ms-overflow-style:none (IE), mais il manque le masquage WebKit pour Chrome/Safari/Edge : ::-webkit-scrollbar { display: none }. La barre de défilement sera visible sur tous les navigateurs Chromium.
Correction : Ajouter un style CSS .review-track::-webkit-scrollbar { display: none }.
Critique Nombre deavis dans JSON-LD incorrect
Ligne 311
"reviewCount": "6" dans le JSON-LD alors que le badge affiche « 25+ reviews » et le carrousel en montre 13. Google pourrait pénaliser cette incohérence.
Correction : Mettre à jour reviewCount et la liste review dans le JSON-LD.
Critique Code mort — fonction initTeamTilt appelée sur des éléments inexistants
Ligne 2518 + 2521-2639
setTimeout(initTeamTilt, 200) est appelé, mais initTeamTilt() cherche des éléments .pc-card-wrapper qui n'existent plus dans le DOM (section équipe supprimée). La fonction fait 120 lignes de calculs inutiles à chaque chargement.
Correction : Supprimer l'appel setTimeout(initTeamTilt, 200) et toute la fonction initTeamTilt().
Critique Sélecteur .pc-hours-row — code mort
Ligne 2328
document.querySelectorAll('.pc-hours-row') ne trouve aucun élément dans le DOM (ancienne section équipe supprimée). Code mort qui s'exécute inutilement.
Correction : Supprimer le bloc .pc-hours-row de la fonction annotateHours().
Critique Texte alternatif des images de la section À propos non traduit
Lignes 1292-1297
Les attributs alt des images du carrousel (« Electro Legendre appliances », « Electro Legendre showroom », etc.) restent en anglais quel que soit le langage choisi.
Correction : Ajouter data-i18n aux balises img ou gérer via JS.

2. Problèmes moyens

Moyen Compteur d'avis incohérent entre le badge et le carrousel
Ligne 649 vs 970
Le badge Google Reviews affiche « Based on 25+ reviews » alors que la clé i18n googleReviewCount dit « 23+ Google Reviews ». Deux chiffres différents sur la même page.
Correction : Unifier le nombre (25+) partout.
Moyen Absence de og:locale:alternate pour le français
Ligne 11
La balise og:locale est en_CA mais il n'y a pas de <meta property="og:locale:alternate" content="fr_CA">. Facebook/LinkedIn ne sauront pas que la page est aussi disponible en français.
Correction : Ajouter <meta property="og:locale:alternate" content="fr_CA">.
Moyen Attribut data-per-page="3" inutilisé sur le carrousel d'avis
Ligne 978
L'attribut data-per-page="3" est défini mais jamais lu par le JavaScript. Les boutons prev/next scrolilent d'une seule diapositive, pas de 3.
Correction : Soit implémenter la logique pagination, soit supprimer l'attribut.
Moyen Le menu mobile se ferme sur mousedown au lieu de click
Ligne 2443
L'événement mousedown ferme le menu avant que le clic n'atteigne sa cible. Si un utilisateur clique sur un lien en arrière-plan, le menu se ferme mais le lien peut aussi être cliqué.
Correction : Utiliser click ou vérifier e.target.closest('a').
Moyen Le menu en cascade est à l'intérieur de <main>
Ligne 548 vs 614
Le staggered-menu-wrapper est placé à l'intérieur de <main id="main-content">, mais c'est un overlay de navigation qui devrait logiquement être en dehors de <main>. Les lecteurs d'écran pourraient le considérer comme du contenu principal.
Correction : Déplacer le menu avant <main>.
Moyen Le popup WhatsApp n'empêche pas le défilement de la page
Ligne 2756
closeWaPopup réinitialise document.body.style.overflow, mais l'ouverture du popup WhatsApp ne le définit pas à 'hidden'. L'utilisateur peut défiler l'arrière-plan derrière le popup.
Correction : Ajouter document.body.style.overflow = 'hidden' lors de l'ouverture du popup WhatsApp.
Moyen Les images de crédit dans la section À propos peuvent ne pas se charger
Lignes 1306-1310
Les URLs externes (Pinterest, Samsung, AJ Madison, Home Depot) sont des liens directs qui peuvent être bloqués par des politiques CORS ou devenir invalides. Aucun fallback onerror n'est défini sur ces images.
Correction : Ajouter onerror avec un fallback, ou héberger les images localement.
Moyen Le bouton WhatsApp flottant peut chevaucher la barre CTA mobile
Lignes 1799-1812
Sur mobile, le bouton WhatsApp est à bottom-20 (80px) et la barre d'appel est à bottom-4 (16px). Avec la hauteur de la barre (~48px), l'espace entre les deux est d'environ 16px. Sur de très petits écrans (< 375px), ils pourraient se chevaucher.
Correction : Vérifier sur iPhone SE / Galaxy Mini et ajuster les positions si nécessaire.
Moyen Les boutons prev/next du carrousel d'avis débordent sur mobile
Lignes 1267-1268
Les boutons utilisent -translate-x-3 / translate-x-3 pour se positionner en dehors du conteneur. Sur mobile, ils peuvent dépasser de l'écran ou être difficiles à atteindre.
Correction : Masquer les boutons sur mobile (hidden md:flex) ou les repositionner à l'intérieur.
Moyen Le label du formulaire utilise sr-only mais le placeholder est visible
Lignes 1392-1393
Les labels sont cachés visuellement (sr-only) mais les placeholders servent de labels visuels. Les placeholders disparaissent quand l'utilisateur tape, ce qui peut causer une confusion sur ce que le champ attend.
Correction : Utiliser des labels flottants ou des placeholders persistants.
Moyen La validation du formulaire ne valide pas au premier focus
Ligne 2268-2271
La validation ne se déclenche que sur l'événement input. Si l'utilisateur clique directement sur « Envoyer » sans toucher aux champs, les erreurs ne s'afficheront que si le formulaire est soumis. Les champs vides ne montrent pas d'erreur tant qu'on n'a pas interagi avec eux.
Correction : Ajouter validation sur blur pour montrer les erreurs plus tôt.
Moyen La classe pe-16 sur le textarea dépend du CDN Tailwind
Ligne 1410
La classe pe-16 (padding-inline-end) n'est définie que dans le CDN Tailwind. Si le CDN est lent ou bloqué, le compteur de caractères chevauchera le texte saisi.
Correction : Définir pe-16 dans le bloc <style> local comme fallback.
Moyen Les images de la section À propos utilisent loading="lazy" sur toutes les slides
Lignes 1292-1297
La première image du carrousel devrait avoir loading="eager" pour s'afficher immédiatement. Toutes les slides being lazy peut retarder l'affichage de la première image.
Correction : Mettre loading="eager" sur la première slide, garder lazy sur les autres.

3. Améliorations suggérées

Amélioration Ajouter un favicon PNG 32x32 en plus du SVG
Le favicon actuel est un SVG data URI. Certains anciens navigateurs et la barre d'onglets ne l'affichent pas correctement. Un PNG 32x32 en plus améliorerait la compatibilité.
Amélioration Supprimer les clés i18n mortes dans les dictionnaires
Lignes 1824, 1874-1879, 1833-1834, etc.
Les clés team, meetTeam, teamTitle, teamDesc, borrelTitle, tonyTitle, available, idx13, idx14, idx16-30, idx89 existent dans les dictionnaires mais ne sont plus utilisées dans le HTML (section équipe supprimée, section produits supprimée). Environ 25 clés mortes.
Amélioration Ajouter rel="dns-prefetch" pour les domains externes
Les ressources proviennent de cdn.tailwindcss.com, cdnjs.cloudflare.com, fonts.googleapis.com, commons.wikimedia.org, i.imgur.com, i.pinimg.com. Ajouter <link rel="dns-prefetch"> pour les domains non préconnectés améliorerait le temps de chargement.
Amélioration Le lien canonical pointe vers electrolegendre.com
Ligne 18
Le domaine electrolegendre.com doit être vérifié. Si le site est hébergé ailleurs (ex: GitHub Pages), le lien canonical créerait un conflit SEO.
Amélioration Le JSON-LD utilise un hasMerchantReturnPolicy avec null
Ligne 339-341
"merchantReturnDays": null pourrait causer une erreur de validation Google. Mieux vaut supprimer cette propriété ou mettre un nombre.
Amélioration Les images imgur n'ont pas de width et height
Ligne 621
L'image hero n'a pas d'attributs width et height. Google Lighthouse recommande ces attributs pour éviter les décalages de mise en page (CLS).
Amélioration Ajouter un sitemap.xml
Pour un site one-page, un sitemap.xml avec l'URL unique et les ancres principales améliorerait l'indexation Google.
Amélioration Les erreurs de validation du formulaire ne sont pas accessibles aux lecteurs d'écran
Les messages d'erreur utilisent role="alert" et aria-describedby ce qui est bien, mais l'absence de aria-live="assertive" sur les conteneurs d'erreur pourrait empêcher l'annonce automatique des erreurs.
